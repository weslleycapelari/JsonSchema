# Schema2DDL

`Schema2DDL` is a **Schema-to-DDL** utility designed to read JSON Schema documents and generate relational database **DDL (Data Definition Language)** SQL scripts for creating tables, columns, constraints, and relationships.

It supports multiple SQL dialects (PostgreSQL, Firebird, SQLite, and SQL Server) and provides both a command-line interface (CLI) for DevOps/automation pipelines and a modern visual VCL desktop application.

## Features

- **Multi-Dialect Code Generation**: Generate customized SQL scripts for PostgreSQL, Firebird, SQLite, and SQL Server.
- **Relational Mapping**: Maps JSON Schema object types and array-of-objects to relational child tables linked by primary and foreign keys automatically.
- **Validation Constraints Mapping**: Translates JSON Schema validation keywords (like `required`, `default`, `maxLength`) directly to SQL constraints (`NOT NULL`, `DEFAULT`, `VARCHAR(N)`).
- **Auto-Increment Support**: Automatically configures primary keys with DB-specific auto-increment clauses (e.g., `SERIAL` in PG, `GENERATED BY DEFAULT AS IDENTITY` in Firebird, `AUTOINCREMENT` in SQLite).
