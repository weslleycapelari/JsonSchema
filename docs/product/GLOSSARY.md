# Glossary

## Anchor

A named location inside a schema document that can be resolved by reference.

## Base URI

The URI used as the starting point when resolving relative references.

## Boolean schema

A schema represented by the JSON literals `true` or `false`.

## Compiled schema

The runtime object produced from a parsed schema, containing keyword validators.

## Context

Structured technical data attached to a validation error.

## Draft

A version of the JSON Schema specification.

## Dialect

The library representation of a draft.

## Instance

The JSON value being validated.

## Keyword

A JSON Schema rule such as `type`, `minimum`, or `$ref`.

## Locale

The language used to render validation messages.

## Registry

The lookup structure used to resolve schemas and references by URI.

## Schema

A JSON document that describes constraints for instances.

## Sub-schema

A nested schema inside another schema keyword.

## Validation result

The result returned after validation, including `IsValid` and the collected errors.

## Vocabulary

A group of related keywords defined by the JSON Schema specification.
